# 可解释旅行规划 Agent 平台

**技术栈：**Python、FastAPI、LangGraph、React、TypeScript、PostgreSQL、mem0、MCP、Langfuse、ChromaDB、Elasticsearch

**项目描述：**  
面向旅行规划场景的可解释 Agent 平台，支持用户发起行程需求并自动完成多轮规划。系统以 LangGraph 状态机为主干，串联偏好收集、约束归一、目的地检索、路线规划、天气查询、预算估算、行程合成与安全审查等节点，并结合上下文工程、记忆机制、harness 工程和工具调用控制提升规划稳定性与可追踪性。

**项目职责：**
- 设计旅行规划主流程为 **workflow-based agent**，基于 LangGraph 将旅行规划拆解为可编排节点与条件分支，使规划过程可解释、可回放、可扩展。
- 基于 **context engineering** 设计上下文装配与压缩策略，将系统提示、任务上下文、最近对话、工具结果和记忆信息按优先级动态组织，减少无效历史对上下文窗口的占用。
- 基于 **mem0 + PostgreSQL** 构建记忆能力，mem0 负责跨会话偏好和画像沉淀，PostgreSQL 负责会话历史持久化，支持会话切换、历史回放和追问续接。
- 设计 **harness 工程** 对工具调用进行统一封装，整合超时、重试、权限控制、失败降级和风险审查，保证外部能力接入时主流程稳定可控。
- 接入 **MCP** 作为工具与外部能力入口，统一承接地图、地理编码、路线规划和天气相关能力，并通过本地兜底保证流程连续性。
- 通过 **Langfuse** 打通 session 级观测链路，将状态迁移、工具调用和模型输出串联起来，支持问题定位、路径回放和质量分析。
- 设计 **工具调用控制与安全审查**，对外部 API 调用引入超时、重试、权限边界和风险兜底，降低高风险工具对主流程的影响。

**可写成简历的一版：**  
> 负责搭建可解释旅行规划 Agent 平台，基于 LangGraph 构建旅行规划 workflow，将偏好收集、约束归一、目的地检索、路线规划、天气查询、预算估算、行程合成与安全审查拆解为可编排节点；基于 context engineering 设计上下文装配与压缩策略，结合系统提示、任务上下文、最近对话、工具结果和记忆信息控制 token 成本；结合 mem0 与 PostgreSQL 构建记忆能力，支持跨会话偏好沉淀、会话历史持久化和历史回放；通过 harness 工程对工具调用进行统一封装，整合超时、重试、权限控制和失败降级；接入 MCP 与 Langfuse 打通工具调用和观测链路，提升系统可解释性与可运维性。

**更适合面试表达的亮点：**
- 不是把 Agent 做成一个聊天壳，而是把它做成可解释、可回放、可观测的执行系统。
- 记忆不是简单存文本，而是围绕会话历史和跨会话偏好做统一管理。
- 上下文不是一次性塞满，而是按优先级装配、按阈值压缩、按需摘要。
- MCP 不是为了追概念，而是为了把工具接入做成统一入口。
- 当前主干仍然是 workflow agent，但已经把后续升级到更强 agent 范式需要的 memory、harness 和 eval 基座搭好了。
