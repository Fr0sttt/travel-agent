# 简历面试教程总索引

这组文档是按简历里的能力点单独拆开的“面试讲述版教程”。
它们不是当前代码的逐行说明，而是按目标架构和理想落地方式来写，方便你面试时从“业务问题 -> 架构方案 -> 工程实现 -> 验证效果”完整展开。

## 使用方式

建议你每个点都按同一套话术练：

1. 先用 30 秒说清楚它解决什么问题
2. 再用 1 分钟讲整体方案
3. 再拆成 3 - 5 个关键实现点
4. 最后补一个验证方式和失败兜底

## 讲述原则

- 先讲业务，再讲技术
- 先讲问题，再讲架构
- 先讲约束，再讲性能
- 先讲可验证性，再讲“看起来高级”

## 建议顺序

如果你是准备面试，推荐按下面顺序记：

1. 上下文工程
2. 记忆分层
3. 意图识别
4. 状态机编排
5. 评测与自进化
6. 安全防护

这样叙事会比较顺：从“怎么把输入管住”，讲到“怎么记住人”，再讲“怎么决定做什么”，最后讲“怎么验证和保护”。
